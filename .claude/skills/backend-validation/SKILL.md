# backend-validation

## Description
Input validation and sanitization patterns for backend APIs using schema validation and business rule enforcement.

## Core Principles
1. **Fail Fast**: Validate early, reject invalid input immediately.
2. **Schema First**: Define schemas before implementing handlers.
3. **Contextual Validation**: Validate based on business context, not just types.
4. **Sanitize Output**: Always escape/sanitize data before rendering.
